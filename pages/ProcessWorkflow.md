# Swimlane Workflow
..

## New
Item state New
 The item is newly added to the board
 Description and acceptance criteria is clear

## Blocked
Item state New
 The item is blocked due to impediment

## Active
Item state Active
 The item is being implemented by a developer

## Ready for testing
Item state Active
 The item is done by the developer
 The item is ready for testing by a QE

## In testing
Item state Active
 The item is picked by a QE
 Testing is performed

## Resolved
Item state Resolved
 Testing is completed
 The item is ready to be accepted by the business

## Closed
Item state Closed
 The item is accepted by the business
 Functionality is dropped and will not be part of the development effort anymore
